1. أساسيات البرمجة ومقدمة في C#

- ما هي البرمجة ولماذا تختار C#؟

البرمجة هي كتابة مجموعة من التعليمات (Code) يقوم الحاسوب بتنفيذها لحل مشكلة معينة أو تحقيق هدف. تخيلها كأنك تعلم الكمبيوتر كيف يقوم بمهمة ما خطوة بخطوة.

لماذا تختار C#؟

  1. شاملة وسهلة الاستخدام: يمكن استخدامها لتطوير برامج سطح المكتب، تطبيقات الهواتف، المواقع الإلكترونية، وحتى الألعاب.
  2. مكتبات قوية: تحتوي على أدوات ومكتبات جاهزة تساعدك على أداء مهام معقدة بسهولة.
  3. مدعومة من مايكروسوفت: وبالتالي تضمن تحديثات مستمرة ودعم واسع.
  4. قابلية التطوير: تُستخدم لتطوير مشاريع صغيرة وكبيرة مثل الأنظمة المؤسسية الكبيرة.

- أدوات البرمجة (IDE)

للكتابة وتنفيذ كود C#، تحتاج إلى بيئة تطوير:

  1. Visual Studio: أفضل أداة تطوير لـC#. تدعم كتابة الكود وتصحيحه وبناء المشاريع بسهولة.
    • تحميله: من موقع مايكروسوفت الرسمي.
    • نسخته المجانية: Visual Studio Community.
  2. Visual Studio Code: محرر خفيف يدعم C# مع إضافة التوسعات المناسبة مثل C# Extension.
    • مناسب أكثر للمشاريع الصغيرة أو البسيطة.

- كتابة أول برنامج (Hello World)

هذا هو أول كود ستكتبه وتنفذه في C#. الهدف منه تعريفك ببنية البرنامج.
الكود:

csharp
using System; // مكتبة النظام الأساسية

class Program // تعريف الفئة الرئيسية
{
    static void Main() // نقطة الدخول الرئيسية للبرنامج
    {
        Console.WriteLine("Hello, World!"); // طباعة النص على الشاشة
    }
}
شرح الكود:
  1. using System;: تضمين مكتبة النظام التي تحتوي على وظائف مثل الطباعة على الشاشة.
  2. class Program: تعريف فئة (Class) باسم "Program". البرنامج يدور داخل هذه الفئة.
  3. static void Main(): الدالة الرئيسية التي يبدأ منها تنفيذ أي برنامج في C#.
  4. Console.WriteLine: أمر طباعة النص على الشاشة.
  5. الناتج:
    Hello, World!
تنفيذ الكود:
  1. افتح Visual Studio.
  2. أنشئ مشروع Console Application جديد.
  3. انسخ الكود أعلاه في ملف Program.cs.
  4. اضغط على زر Run أو F5.

2. التحكم في تدفق البرنامج

التحكم في تدفق البرنامج يعني تحديد كيف يتصرف البرنامج بناءً على الشروط أو الحلقات. هنا نبدأ بشرح الجمل الشرطية والحلقات التكرارية.

الجمل الشرطية (Conditional Statements)

الجمل الشرطية تسمح لك باتخاذ قرارات بناءً على شروط معينة.

- if Statement:

إذا تحقق الشرط، ينفذ الكود داخل القوسين {}.

csharp
int age = 18;

if (age >= 18)
{
    Console.WriteLine("You are eligible to vote."); // يتم تنفيذ هذا فقط إذا كان العمر أكبر أو يساوي 18
}
- if-else Statement:

تضيف خيارًا ثانيًا إذا لم يتحقق الشرط.

csharp
int age = 16;

if (age >= 18)
{
    Console.WriteLine("You are eligible to vote.");
}
else
{
    Console.WriteLine("You are not eligible to vote.");
}
- if-else if-else Statement:

تسمح بإضافة عدة شروط.

csharp
int marks = 85;

if (marks >= 90)
{
    Console.WriteLine("Grade: A");
}
else if (marks >= 75)
{
    Console.WriteLine("Grade: B");
}
else
{
    Console.WriteLine("Grade: C");
}
- switch Statement:

مناسبة عند وجود خيارات متعددة.

csharp
int day = 3;

switch (day)
{
    case 1:
        Console.WriteLine("Monday");
        break;
    case 2:
        Console.WriteLine("Tuesday");
        break;
    case 3:
        Console.WriteLine("Wednesday");
        break;
    default:
        Console.WriteLine("Invalid day");
        break;
}
  • break: يخرج من switch بعد تنفيذ الحالة المناسبة.
  • default: ينفذ إذا لم يتطابق أي خيار.

الحلقات التكرارية (Loops)

تستخدم لتكرار تنفيذ الكود حتى يتحقق شرط معين أو ينتهي.

- for Loop:

تكرار الكود عددًا محددًا من المرات.

csharp
for (int i = 1; i <= 5; i++) // يبدأ i من 1 ويستمر حتى 5
{
    Console.WriteLine("Number: " + i);
}

الناتج:

javascript
Number: 1
Number: 2
Number: 3
Number: 4
Number: 5
- while Loop:

يستمر في التكرار طالما الشرط صحيح.

csharp
int i = 1;

while (i <= 5)
{
    Console.WriteLine("Number: " + i);
    i++; // زيادة i بعد كل تكرار
}
- do-while Loop:

يشبه while، لكن يتم تنفيذ الكود مرة واحدة على الأقل حتى لو كان الشرط خاطئًا.

csharp
int i = 6;

do
{
    Console.WriteLine("Number: " + i);
    i++;
} while (i <= 5);

الناتج:

javascript
Number: 6
- foreach Loop:

تُستخدم لتكرار العناصر في مجموعة (مثل مصفوفة أو قائمة).

csharp
int[] numbers = { 1, 2, 3, 4 };

foreach (int number in numbers)
{
    Console.WriteLine("Number: " + number);
}

الناتج:

javascript
Number: 1
Number: 2
Number: 3
Number: 4

3. المصفوفات (Arrays) والمجموعات (Collections)

- المصفوفات (Arrays)

المصفوفة هي مجموعة من العناصر ذات النوع نفسه يتم تخزينها بترتيب متتالي. يمكنك تخزين البيانات فيها واستخدامها بسهولة.

- تعريف مصفوفة:
csharp
int[] numbers = { 10, 20, 30, 40 };
Console.WriteLine(numbers[0]); // Output: 10
Console.WriteLine(numbers[2]); // Output: 30
  • numbers[0]: تشير إلى العنصر الأول في المصفوفة.
  • يمكن الوصول للعناصر باستخدام الفهارس (Indexes) التي تبدأ من 0.
- مصفوفة متعددة الأبعاد:
csharp
int[,] matrix = {
    { 1, 2, 3 },
    { 4, 5, 6 },
    { 7, 8, 9 }
};

Console.WriteLine(matrix[1, 1]); // Output: 5

- القوائم (Lists)

القائمة هي مجموعة ديناميكية يمكن تعديلها (إضافة/حذف عناصر).

- إنشاء قائمة وإضافة عناصر:
csharp
List<string> names = new List<string>();
names.Add("Alice");
names.Add("Bob");
Console.WriteLine(names[0]); // Output: Alice
- إزالة عناصر من القائمة:
csharp
names.Remove("Alice");
Console.WriteLine(names.Count); // Output: 1
- التكرار باستخدام foreach:
csharp
foreach (string name in names)
{
    Console.WriteLine(name);
}

4. الدوال (Functions)

الدوال في البرمجة هي عبارة عن مجموعة من التعليمات التي تنفذ مهمة محددة.
استخدام الدوال يجعل الكود منظمًا وقابلًا لإعادة الاستخدام.

ما هي الدالة؟

  • الدالة تأخذ مدخلات (اختيارية).
  • تنفذ مجموعة أوامر.
  • تُرجع نتيجة (اختياري).
مثال بسيط:
csharp
static void Greet()
{
    Console.WriteLine("Hello, World!");
}

// استدعاء الدالة
Greet();

الناتج:

Hello, World!

أنواع الدوال

- 1. دوال بدون مدخلات أو مخرجات:

لا تأخذ أي بيانات ولا ترجع أي نتيجة.

csharp
static void SayHello()
{
    Console.WriteLine("Hello!");
}

SayHello(); // استدعاء الدالة
- 2. دوال تأخذ مدخلات:

تأخذ بيانات عند استدعائها وتستخدمها.

csharp
static void Greet(string name)
{
    Console.WriteLine("Hello, " + name + "!");
}

Greet("Ali"); // Output: Hello, Ali!
- 3. دوال تُرجع قيمة:

ترجع نتيجة بعد تنفيذ الأوامر.

csharp
static int Add(int a, int b)
{
    return a + b;
}

int result = Add(5, 3);
Console.WriteLine(result); // Output: 8
- 4. دوال تأخذ مدخلات وتُرجع قيمة:

تجمع بين النوعين السابقين.

csharp
static double CalculateArea(double radius)
{
    return 3.14 * radius * radius;
}

double area = CalculateArea(5);
Console.WriteLine(area); // Output: 78.5

Overloading (تحميل زائد للدوال)

في C#، يمكنك تعريف أكثر من دالة بنفس الاسم بشرط أن تختلف في:

  1. عدد المعاملات.
  2. نوع المعاملات.
مثال:
csharp
static void Print(int number)
{
    Console.WriteLine("Number: " + number);
}

static void Print(string text)
{
    Console.WriteLine("Text: " + text);
}

Print(10);       // Output: Number: 10
Print("Hello");  // Output: Text: Hello

Pass by Value vs Pass by Reference

- Pass by Value:

يتم إرسال نسخة من القيمة إلى الدالة.

csharp
static void Increment(int num)
{
    num++;
}

int value = 5;
Increment(value);
Console.WriteLine(value); // Output: 5 (القيمة الأصلية لم تتغير)
- Pass by Reference:

يتم إرسال المرجع، وبالتالي يمكن تعديل القيمة الأصلية.

csharp
static void Increment(ref int num)
{
    num++;
}

int value = 5;
Increment(ref value);
Console.WriteLine(value); // Output: 6 (القيمة الأصلية تغيرت)

5. البرمجة الكائنية (OOP - Object-Oriented Programming)

ما هي البرمجة الكائنية؟

OOP هي طريقة لتنظيم الكود بتقسيمه إلى كائنات (Objects)، وكل كائن يحتوي على:

  1. الخصائص (Properties): تمثل البيانات.
  2. الطرق (Methods): تمثل الأفعال.

1. الكائنات (Objects) والفئات (Classes)

- تعريف الفئة:
csharp
class Car
{
    public string Brand;
    public void Drive()
    {
        Console.WriteLine(Brand + " is driving.");
    }
}
- إنشاء كائن واستخدامه:
csharp
Car myCar = new Car();
myCar.Brand = "Toyota";
myCar.Drive(); // Output: Toyota is driving.

2. الإنشاء (Constructors)

الدوال الإنشائية تُستخدم لتخصيص قيم الكائن عند إنشائه.

csharp
class Car
{
    public string Brand;

    // Constructor
    public Car(string brand)
    {
        Brand = brand;
    }

    public void Drive()
    {
        Console.WriteLine(Brand + " is driving.");
    }
}

Car myCar = new Car("Honda");
myCar.Drive(); // Output: Honda is driving.

3. الوراثة (Inheritance)

تتيح لك إعادة استخدام الكود من فئة أخرى.

csharp
class Vehicle
{
    public void Move()
    {
        Console.WriteLine("The vehicle is moving.");
    }
}

class Car : Vehicle
{
    public void Honk()
    {
        Console.WriteLine("The car is honking.");
    }
}

Car myCar = new Car();
myCar.Move(); // Output: The vehicle is moving.
myCar.Honk(); // Output: The car is honking.

4. التغليف (Encapsulation)

إخفاء التفاصيل الداخلية للفئة وجعلها آمنة باستخدام private وpublic.

csharp
class Person
{
    private string name;

    public void SetName(string newName)
    {
        name = newName;
    }

    public string GetName()
    {
        return name;
    }
}

Person person = new Person();
person.SetName("Ali");
Console.WriteLine(person.GetName()); // Output: Ali

5. التعددية (Polymorphism)

إعادة تعريف الدوال في الفئات الموروثة باستخدام override.

csharp
class Animal
{
    public virtual void Sound()
    {
        Console.WriteLine("Animal makes a sound.");
    }
}

class Dog : Animal
{
    public override void Sound()
    {
        Console.WriteLine("Dog barks.");
    }
}

Animal myDog = new Dog();
myDog.Sound(); // Output: Dog barks.

6. التعامل مع الأخطاء (Exception Handling)

التعامل مع الأخطاء هو جزء أساسي من البرمجة لضمان أن البرنامج يستمر في العمل حتى في حالة حدوث خطأ غير متوقع.

في C#، يتم التعامل مع الأخطاء باستخدام try-catch-finally. دعنا نتعلم كيف يعمل ذلك خطوة بخطوة.

لماذا نحتاج التعامل مع الأخطاء؟

  • لمنع انهيار البرنامج عند حدوث أخطاء (مثل تقسيم عدد على صفر).
  • لتقديم رسائل مفيدة للمستخدم.
  • لضمان أن بعض الإجراءات تُنفذ دائمًا، مثل إغلاق ملف أو تحرير ذاكرة.

البنية الأساسية لـ try-catch

csharp
try
{
    // الكود الذي قد يحتوي على خطأ
}
catch (Exception ex)
{
    // التعامل مع الخطأ
}
finally
{
    // كود يتم تنفيذه دائمًا (اختياري)
}

مثال عملي: تقسيم عدد على صفر

csharp
try
{
    int result = 10 / 0; // هذا سيؤدي إلى خطأ
}
catch (DivideByZeroException ex)
{
    Console.WriteLine("Error: Cannot divide by zero."); // التعامل مع الخطأ
}
finally
{
    Console.WriteLine("Execution completed."); // يتم تنفيذ هذا الجزء دائمًا
}

الناتج:

vbnet
Error: Cannot divide by zero.
Execution completed.
- try: يحتوي على الكود الذي قد يُسبب خطأ.
- catch: يتم تنفيذ هذا الجزء إذا وقع خطأ.
- finally: يتم تنفيذه دائمًا، سواء حدث خطأ أم لا.

أنواع الأخطاء (Exceptions)

C# توفر عدة أنواع من الأخطاء. بعض الأمثلة:

  1. DivideByZeroException: عند محاولة القسمة على صفر.
  2. NullReferenceException: عند محاولة استخدام كائن غير مُعرف (null).
  3. IndexOutOfRangeException: عند محاولة الوصول إلى عنصر غير موجود في مصفوفة أو قائمة.
  4. FileNotFoundException: إذا كان الملف المطلوب غير موجود.

تعامل مع أخطاء محددة

يمكنك كتابة عدة كتل catch للتعامل مع أنواع مختلفة من الأخطاء.

csharp
try
{
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]); // محاولة الوصول إلى فهرس غير موجود
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("Error: Index out of range.");
}
catch (Exception ex)
{
    Console.WriteLine("An error occurred: " + ex.Message);
}
finally
{
    Console.WriteLine("Execution finished.");
}

الناتج:

csharp
Error: Index out of range.
Execution finished.

إلقاء الأخطاء يدويًا (Throw Exceptions)

يمكنك استخدام throw لإلقاء خطأ بنفسك.

csharp
static void CheckAge(int age)
{
    if (age < 18)
    {
        throw new ArgumentException("Age must be 18 or older.");
    }
}

try
{
    CheckAge(16);
}
catch (ArgumentException ex)
{
    Console.WriteLine("Error: " + ex.Message);
}

الناتج:

javascript
Error: Age must be 18 or older.

إنشاء نوع مخصص من الأخطاء (Custom Exception)

إذا كنت تحتاج لخطأ خاص بمشروعك.

csharp
class CustomException : Exception
{
    public CustomException(string message) : base(message) { }
}

try
{
    throw new CustomException("This is a custom exception.");
}
catch (CustomException ex)
{
    Console.WriteLine("Custom error: " + ex.Message);
}

الناتج:

vbnet
Custom error: This is a custom exception.

أفضل ممارسات التعامل مع الأخطاء

  1. تعامل مع الأخطاء المحددة فقط: لا تستخدم catch (Exception ex) إلا إذا كان ضروريًا.
  2. اجعل رسائل الخطأ واضحة: لتسهيل التصحيح.
  3. استخدم finally للموارد المهمة: مثل إغلاق الملفات أو تحرير الاتصال بقواعد البيانات.
  4. لا تعتمد على الأخطاء كجزء من منطق البرنامج.

7. العمل مع الملفات (File Handling)

التعامل مع الملفات هو مهارة أساسية عند إنشاء برامج تحتاج إلى تخزين أو قراءة بيانات من وإلى نظام الملفات. في C#، يتم ذلك باستخدام مكتبة System.IO.

1. أهم الكائنات في مكتبة System.IO

  1. File: لتوفير وظائف شائعة للعمل مع الملفات (قراءة، كتابة، حذف).
  2. FileStream: لتوفير طريقة أكثر تحكمًا عند قراءة أو كتابة الملفات.
  3. StreamReader و StreamWriter: للقراءة والكتابة على النصوص.
  4. Directory: للعمل مع المجلدات.

2. كتابة النصوص إلى ملف

- باستخدام File.WriteAllText:

يكتب النصوص مباشرة إلى ملف. إذا لم يكن الملف موجودًا، يتم إنشاؤه.

csharp
using System.IO;

string path = "example.txt";
string content = "Hello, this is a sample text.";

File.WriteAllText(path, content);
Console.WriteLine("File written successfully.");
الناتج:

يتم إنشاء ملف باسم example.txt يحتوي على النص:

kotlin
Hello, this is a sample text.

3. قراءة النصوص من ملف

- باستخدام File.ReadAllText:

يقرأ محتوى ملف نصي دفعة واحدة.

csharp
string path = "example.txt";

if (File.Exists(path))
{
    string content = File.ReadAllText(path);
    Console.WriteLine("File content: " + content);
}
else
{
    Console.WriteLine("File does not exist.");
}
الناتج (إذا كان الملف موجودًا):
arduino
File content: Hello, this is a sample text.

4. الكتابة السطرية إلى ملف

- باستخدام StreamWriter:

للكتابة سطرًا بسطر إلى ملف.

csharp
using (StreamWriter writer = new StreamWriter("example.txt"))
{
    writer.WriteLine("First line.");
    writer.WriteLine("Second line.");
    writer.WriteLine("Third line.");
}

Console.WriteLine("Lines written to file.");

5. قراءة النصوص سطرًا بسطر

- باستخدام StreamReader:

لقراءة محتوى ملف سطرًا بسطر.

csharp
using (StreamReader reader = new StreamReader("example.txt"))
{
    string line;
    while ((line = reader.ReadLine()) != null)
    {
        Console.WriteLine(line);
    }
}
الناتج:
arduino
First line.
Second line.
Third line.

6. إضافة نصوص إلى ملف موجود

- باستخدام File.AppendAllText:

يُضيف نصوصًا إلى ملف موجود.

csharp
string additionalContent = "\nThis is an additional line.";
File.AppendAllText("example.txt", additionalContent);

Console.WriteLine("Text appended to file.");

7. حذف ملف

- باستخدام File.Delete:
csharp
string path = "example.txt";

if (File.Exists(path))
{
    File.Delete(path);
    Console.WriteLine("File deleted successfully.");
}
else
{
    Console.WriteLine("File does not exist.");
}

8. إنشاء مجلدات والعمل معها

- إنشاء مجلد:
csharp
string folderPath = "ExampleFolder";

if (!Directory.Exists(folderPath))
{
    Directory.CreateDirectory(folderPath);
    Console.WriteLine("Folder created successfully.");
}
- حذف مجلد:
csharp
if (Directory.Exists(folderPath))
{
    Directory.Delete(folderPath, true); // true: لحذف المجلد ومحتوياته
    Console.WriteLine("Folder deleted successfully.");
}

9. الحصول على قائمة بالملفات أو المجلدات

- قائمة الملفات:
csharp
string[] files = Directory.GetFiles("C:\\ExampleFolder");

foreach (string file in files)
{
    Console.WriteLine("File: " + file);
}
- قائمة المجلدات:
csharp
string[] directories = Directory.GetDirectories("C:\\ExampleFolder");

foreach (string directory in directories)
{
    Console.WriteLine("Directory: " + directory);
}

10. التعامل مع الأخطاء في File Handling

من الضروري التعامل مع الأخطاء عند العمل مع الملفات لأن العمليات قد تفشل (مثل عدم وجود الملف أو عدم وجود إذن).

csharp
try
{
    string content = File.ReadAllText("nonexistent.txt");
    Console.WriteLine(content);
}
catch (FileNotFoundException ex)
{
    Console.WriteLine("Error: File not found.");

1. ما هي ADO.NET؟

2. الاتصال بقاعدة بيانات SQL Server

خطوات الاتصال:

  1. إضافة مكتبة System.Data.SqlClient.
  2. كتابة سلسلة الاتصال (Connection String).
  3. استخدام SqlConnection لفتح وإغلاق الاتصال.

مثال على الاتصال:

csharp
using System;

{

9. البرمجة متعددة الخيوط (Multithreading)

البرمجة متعددة الخيوط (Multithreading) تُستخدم لتنفيذ مهام متعددة في نفس الوقت داخل تطبيق واحد. هذا يجعل التطبيقات أكثر كفاءة وسرعة، خاصة عند التعامل مع العمليات الطويلة مثل معالجة البيانات الكبيرة أو تحميل الملفات.

1. ما هي الخيوط (Threads)؟

الخيط هو مسار منفصل يمكن للتطبيق تنفيذه بشكل متزامن مع المسارات الأخرى.
في C#، يمكن استخدام مكتبة System.Threading لإنشاء وإدارة الخيوط.

2. إنشاء خيط بسيط

باستخدام Thread

csharp
using System;
using System.Threading;

class Program
{
    static void PrintNumbers()
    {
        for (int i = 1; i <= 5; i++)
        {
            Console.WriteLine("Number: " + i);
            Thread.Sleep(1000); // تأخير لمدة ثانية واحدة
        }
    }

    static void Main()
    {
        Thread thread = new Thread(PrintNumbers);
        thread.Start();

        Console.WriteLine("Main thread is running.");
    }
}
شرح:
  • Thread: لإنشاء وإدارة الخيوط.
  • Thread.Sleep: لإيقاف الخيط لفترة زمنية مؤقتة.
  • thread.Start(): لبدء تنفيذ الخيط.

5. مزامنة الخيوط (Thread Synchronization)

لماذا نحتاج المزامنة؟

عند محاولة عدة خيوط الوصول إلى نفس المورد (مثل متغير مشترك)، يمكن أن يحدث سباق الخيوط (Race Condition)، مما يؤدي إلى نتائج غير متوقعة.

استخدام الأقفال (Locks)

csharp
static object lockObject = new object();
static int counter = 0;

static void Increment()
{
    for (int i = 0; i < 5; i++)
    {
        lock (lockObject)
        {
            counter++;
            Console.WriteLine("Counter: " + counter);
        }
        Thread.Sleep(100);
    }
}

static void Main()
{
    Thread t1 = new Thread(Increment);
    Thread t2 = new Thread(Increment);

    t1.Start();
    t2.Start();

    t1.Join();
    t2.Join();

    Console.WriteLine("Final Counter: " + counter);
}

2. بدء التحميل:

csharp
using System.Net.Http;
using System.IO;
using System.Threading.Tasks;

static async Task StartDownload()
{
    if (fileUrls.Count == 0)
    {
        Console.WriteLine("No URLs to download.");
        return;
    }

    Console.WriteLine("\nStarting downloads...");

    List<Task> downloadTasks = new List<Task>();

    foreach (string url in fileUrls)
    {
        downloadTasks.Add(DownloadFileAsync(url));
    }

    await Task.WhenAll(downloadTasks);
    Console.WriteLine("All downloads completed.");
}

static async Task DownloadFileAsync(string url)
{
    try
    {
        using HttpClient client = new HttpClient();
        string fileName = Path.GetFileName(new Uri(url).LocalPath);
        Console.WriteLine($"Downloading {fileName}...");

        byte[] fileData = await client.GetByteArrayAsync(url);
        await File.WriteAllBytesAsync(fileName, fileData);

        Console.WriteLine($"{fileName} downloaded successfully.");
        Log($"Success: {fileName}");
    }
    catch (Exception ex)
    {
        Console.WriteLine($"Error downloading {url}: {ex.Message}");
        Log($"Error: {url} - {ex.Message}");
    }
}

csharp
using System;

class TaskItem
{
    public int Id { get; set; }
    public string Title { get; set; }
    public string Description { get; set; }
    public DateTime Deadline { get; set; }
    public bool IsCompleted { get; set; }

    public override string ToString()
    {
        return $"ID: {Id}\nTitle: {Title}\nDescription: {Description}\nDeadline: {Deadline.ToShortDateString()}\nStatus: {(IsCompleted ? "Completed" : "Pending")}\n";
    }
}
